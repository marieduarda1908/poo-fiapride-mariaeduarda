## Perguntas de Reflexão

### Aula 1 - Classes e Objetos

As classes ajudam a organizar melhor os dados e comportamentos do sistema. Se existissem milhões de usuários, seria impossível controlar tudo usando variáveis soltas no main. A classe permite criar vários objetos reutilizando a mesma estrutura, deixando o código mais limpo, organizado e fácil de manter.

---

### Aula 2 - Métodos

Os métodos ajudam a proteger as regras de negócio do sistema. Alterar valores diretamente poderia causar erros ou inconsistências. Com métodos específicos conseguimos validar os dados antes de alterar os atributos.

---

### Aula 3 - Encapsulamento

Os atributos privados protegem os dados internos do objeto. Os getters permitem visualizar as informações de forma segura sem alterar diretamente os valores. Isso evita modificações indevidas e mantém o sistema protegido.

---

### Aula 4 - Construtores

Nem todos os atributos devem ser alterados livremente. Algumas informações precisam ser protegidas para evitar inconsistências. O uso correto de construtores melhora a organização e segurança do sistema.

---

### Aula 5 - Associação

Utilizar o objeto completo permite acessar todas as informações e comportamentos necessários. Se fosse apenas uma String, o sistema não conseguiria interagir corretamente com os dados do objeto associado.

---

### Aula 6 - Herança

O Java protege os atributos privados para manter o encapsulamento. Assim, as subclasses não alteram diretamente informações importantes da classe mãe, garantindo mais segurança no sistema.

---

### Aula 7 - Polimorfismo

O método precisa existir na classe mãe para que o polimorfismo funcione corretamente. Assim, o Java consegue executar o método adequado dependendo do tipo do objeto utilizado.

---

### Aula 8 - Classes Abstratas

Não faz sentido criar um objeto genérico sem um tipo específico. A classe abstrata impede a criação de objetos incompletos e obriga as subclasses a implementarem comportamentos necessários.

---

### Aula 9 - Interfaces

As interfaces permitem compartilhar contratos sem causar problemas de herança múltipla. Assim, diferentes classes podem implementar os mesmos comportamentos de forma organizada.
